What is Seat Allotment?

A system that manages seat reservations in halls, classrooms, or theaters using a 2D grid structure.

Real-World Applications

  • Movie theater seat booking
  • Classroom seating arrangements
  • Conference room management
  • Transportation (bus/train seating)

The Challenge

Manual seat management is:

  • ‚ùå Error-prone (double bookings)
  • ‚ùå Time-consuming (manual checking)
  • ‚ùå Disorganized (no clear layout)

Solution: Automated system using 2D arrays to track availability in real-time.

Main Features

Display Seats

View current layout with rows and seat numbers. 0 = available, 1 = reserved

Reserve Seat

Book a seat by selecting row and seat number with instant validation

Cancel Reservation

Release a reserved seat with confirmation to prevent accidents

2D Array Architecture

Layout: 4 Rows √ó 10 Seats

int seats[4][10] = {0};

Where:
0 = Seat is available (empty)
1 = Seat is reserved (booked)

Configuration through ROWS and COLS macros makes it scalable to any size.

Core Functions

displaySeats()

Prints formatted seating layout with proper alignment

reserveSeat()

Books a seat after range and availability checks

cancelReservation()

Cancels a reservation after validation

Smart Validation

Checks performed for every operation:

  • ‚úì Row number within 1 to ROWS
  • ‚úì Seat number within 1 to COLS
  • ‚úì Seat not already reserved (for booking)
  • ‚úì Seat is reserved (for cancellation)
  • ‚úì User confirmation for irreversible actions

Invalid inputs trigger helpful error messages.

Execution Flow

  1. Initialize 2D array with all 0s (all seats free)
  2. Display menu with 4 options
  3. Get user choice and validate input
  4. Execute corresponding function:
    • Option 1: Show current layout
    • Option 2: Book seat ‚Üí update array
    • Option 3: Cancel ‚Üí confirm ‚Üí update array
    • Option 4: Exit program
  5. Loop until user exits
